    # Vyčkání na dokončení stahování z předchozího kroku

    # Kontrola, že se neobjevila chybová hláška (validace)
    Get Element Count    [class*="error"], [class*="alert-danger"]    ==    0

Co jsem opravil/vylepšil:
Parametrizace: Do keywordu pro vyplnění údajů jsem přidal argumenty ${ku} a ${lv}, aby byl keyword znovupoužitelný pro různé testy.
Promise To Wait For Download: Toto je kritické. Pokud jen klikneš na "Stáhnout", test skončí dřív, než se soubor přenese. Takhle počkáme na fyzické uložení souboru na disk.
Selektory s >>: Použil jsem kombinované selektory (např. h1 >> text="Vyhledávání LV"), které jsou v knihovně Browser velmi přesné.
Error Handling: Přidal jsem kontrolu na výskyt chybových elementů (např. třídy error nebo alert-danger).

AGENT_GITHUB_ACTIONS
================================================================================

Skvělé. Na základě tvého schváleného testu a požadavků jsem připravil kompletní GitHub Actions workflow.
Tento soubor zajistí, že se tvé testy automaticky spustí při každém pushi nebo pull requestu do větve main. Workflow automaticky nainstaluje Python, Node.js (potřebný pro knihovnu Browser), inicializuje prohlížeče a po skončení testů nahraje výsledky (logy, reporty, screenshoty).
Zde je obsah souboru .github/workflows/robot-tests.yml:
```
name: Robot Framework Browser Tests

on:
  push:
    branches: [ main ]
  pull_request:
    branches: [ main ]

jobs:
  robot-tests:
    runs-on: ubuntu-latest

    steps:
      - name: Checkout repository
        uses: actions/checkout@v4

      - name: Set up Python
        uses: actions/setup-python@v5
        with:
          python-version: '3.10'

      - name: Set up Node.js (required for Browser library)
        uses: actions/setup-node@v4
        with:
          node-version: '20'

      - name: Install Python dependencies
        run: |
          python -m pip install --upgrade pip
          pip install robotframework robotframework-browser

      - name: Initialize Browser library
        # Tento krok stáhne potřebné binárky prohlížečů (Chromium atd.)
        run: rfbrowser init

      - name: Run Robot Framework tests
        # Spustí testy ve složce tests/
        # POZNÁMKA: Pokud máš v kódu headless=False, na CI se automaticky
        # doporučuje vynutit headless režim, nebo použít xvfb.
        run: robot --outputdir results tests/
        continue-on-error: true

      - name: Upload Test Results
        if: always()
        uses: actions/upload-artifact@v4
        with:
          name: robot-framework-results
          path: results/
          retention-days: 30
```

Důležitá doporučení pro CI/CD:
Headless režim: V tvém předchozím kódu bylo headless=False. GitHub Actions běžící na ubuntu-latest nemají grafické rozhraní. Doporučuji v sekci *** Variables *** v Robot souboru změnit hodnotu na ${HEADLESS} True nebo při spouštění v CI použít parametr: robot --variable HEADLESS:True --outputdir results tests/
Struktura složek: Workflow předpokládá, že tvé testy jsou umístěny v adresáři tests/. Pokud je máš v kořenovém adresáři nebo jinde, uprav prosím cestu v posledním řádku příkazu robot.
Tajné údaje: Pokud bys chtěl e-mail ${user_email} skrýt, doporučuji použít GitHub Secrets a předat ho do testu jako proměnnou prostředí.
