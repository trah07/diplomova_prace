ako senior automatizační inženýr jsem revidoval vygenerovaný Robot Framework kód. Provedl jsem několik úprav, aby byl skript robustnější, odpovídal standardům knihovny Browser (Playwright) a reflektoval reálné chování webových aplikací (včetně ošetření asynchronních prvků).
Provedené změny a vylepšení:
Optimalizace selektorů: Použil jsem kombinaci textových selektorů a CSS, které jsou v knihovně Browser stabilnější.

Validace stavu: Místo prostého Wait For Elements State jsem přidal kontrolu specifických atributů (např. aria-invalid nebo přítomnost chybové třídy), což je u testování validity formulářů přesnější.
Ošetření stažení: Doplnil jsem logiku, která explicitně ověřuje, že se neotevřel dialog pro stažení nebo nezačal download stream.

AGENT_GITHUB_ACTIONS
================================================================================

Jako expert na CI/CD jsem připravil konfigurační soubor pro GitHub Actions. Tento workflow automaticky spustí vaše Robot Framework testy při každém pushi nebo pull requestu do větve main.
Konfigurace zahrnuje instalaci Pythonu, Node.js (nezbytný pro knihovnu Browser), inicializaci prohlížečů a následné nahrání výsledků (logy, reporty, screenshoty) jako artefakty, a to i v případě, že testy selžou.
Zde je obsah souboru .github/workflows/robot-tests.yml:

KÓD
--------------------------------------------------------------------------------
name: Robot Framework Tests
on:
push:
branches: [ "main" ]
pull_request:
branches: [ "main" ]
jobs:
test:
name: Run Browser Library Tests
runs-on: ubuntu-latest
steps:
- name: Checkout repository
uses: actions/checkout@v4
- name: Set up Python
uses: actions/setup-python@v5
with:
python-version: '3.10'
cache: 'pip'
- name: Set up Node.js
uses: actions/setup-node@v4
with:
node-version: '20'
- name: Install Python dependencies
run: |
python -m pip install --upgrade pip
pip install robotframework robotframework-browser
- name: Initialize Browser library
run: |
# Instalace prohlížečů a jejich závislostí pro Linux
rfbrowser init
- name: Run Robot Framework tests
run: |
# Spuštění testů - předpokládá se umístění v adresáři tests/
# Pokud máte testy v kořenovém adresáři, změňte na: robot --outputdir results .
robot --outputdir results tests/
continue-on-error: false
- name: Upload test results
if: always()
uses: actions/upload-artifact@v4
with:
name: robot-framework-results
path: results/
retention-days: 30

Klíčové vlastnosti tohoto workflow:
Triggery: Aktivuje se při změnách v hlavní větvi.
Prostředí: Používá ubuntu-latest, což je nejlevnější a nejrychlejší standard pro CI.
Browser Initialization: Příkaz rfbrowser init v rámci GitHub Actions automaticky stáhne binárky Chromium a nastaví potřebné systémové knihovny.
Artefakty: Používáme if: always(), což zaručuje, že i když test TS-KN-03 selže, v záložce "Actions" uvidíte ke stažení results/ s log.html a report.html pro snadný debugging.
Cache: Přidána základní cache pro pip, aby se zrychlilo opakované spouštění workflow.
Předkládám ke schválení. Prosím o vyjádření (OK / NEOK).
