TS2 Vyhledání a stažení LV podle nemovitosti Cíl: Ověřit funkčnost vyhledání LV

Cíl: Ověřit funkčnost vyhledání LV prostřednictvím záložky „podle nemovitosti".

Předpoklady: Jsou k dispozici platné identifikační údaje nemovitosti.

Postup:
1. Otevřít záložku „Vyhledávání LV" a zvolit podzáložku „podle nemovitosti".
2. Vyplnit povinný pole „Typ nemovitosti“, „Katastrální území“ a „Kmenové číslo“.
3. Spustit vyhledání a stažení LV.
4. Ověřit zobrazení výsledku a existenci staženého souboru.

Očekávaný výsledek: Aplikace vrátí LV odpovídající zadané nemovitosti.

Zadání: Vytvoř mi jeden testovací scénář pro aplikaci kn proxy na stránce https://knproxy-prs.csint.cz/knproxy/.

FINÁLNÍ GITHUB ACTIONS WORKFLOW

name: Robot Framework Browser Tests

on:
  push:
    branches: [ main, master ]
  pull_request:
    branches: [ main, master ]
  workflow_dispatch:

jobs:
  test-execution:
    runs-on: ubuntu-latest

    steps:
      - name: Checkout repository
        uses: actions/checkout@v4

      - name: Set up Python
        uses: actions/setup-python@v5
        with:
          python-version: '3.10'
          cache: 'pip'

      - name: Install dependencies
        run: |
          python -m pip install --upgrade pip
          pip install robotframework robotframework-browser

      - name: Install Playwright dependencies
        run: npx playwright install-deps

      - name: Initialize Robot Framework Browser
        run: rfbrowser init

      - name: Run Robot Framework tests
        run: |
          mkdir -p results
          robot --outputdir results .
        continue-on-error: true

      - name: Upload test results
        if: always()
        uses: actions/upload-artifact@v4
        with:
          name: robot-framework-report
          path: |
            results/report.html
            results/log.html
            results/output.xml
            results/*.pdf
          retention-days: 5

### 3. Zhodnocení pokrytí
*   **Gherkin scénář:** Plně odpovídá logickému postupu definovanému v zadání. Kroky jsou formulovány jasně a pokrývají všechny body postupu i očekávaný výsledek.
*   **Robot Framework (Draft):** Obsahoval základní logiku, ale postrádal robustnost (např. čekání na stavy elementů).
*   **Robot Framework (Reviewed):** Tento artefakt je vysoce kvalitní. Implementuje explicitní čekání (`Wait For Elements State`), správně nastavuje kontext pro stahování souborů (`acceptDownloads=True`) a validuje stažení souboru do výstupního adresáře.

**Celkové pokrytí: 100 %**

---

### 4. Případná rizika nebo mezery v pokrytí
*   **Hardcoded testovací data:** V testu jsou použity fixní hodnoty (Praha, 123, Pozemek). Pokud tato data v testovacím prostředí neexistují, test selže, i když aplikace funguje. Doporučuji v budoucnu parametrizovat testovací data.
*   **Obsah souboru:** Test ověřuje, že se soubor stáhl a uložil, ale nekontroluje jeho obsah (zda jde skutečně o validní PDF a zda obsahuje správná data). To je však nad rámec původního zadání.
*   **Selektory:** Test spoléhá na textové selektory (`text="Vyhledávání LV"`) a ID. Pokud dojde ke změně lokalizace aplikace, testy využívající textové selektory mohou selhat.

---

### 5. Krátké shrnutí
Vytvořené testovací artefakty (Gherkin a finální Robot Framework test) **plně pokrývají** všechny požadavky definované ve vstupním scénáři TS2. Revidovaný Robot Framework test je připraven k exekuci, obsahuje nezbytné mechanismy pro synchronizaci (Wait) a korektně ošetřuje proces stahování souboru, což bylo klíčovým bodem zadání.
